## PM Release Brief
- Objective: Make usage logs explain how users actually drive the loop.
- Target user value: Operators and kit authors can see session-level behavior, failures, and suspicious release gaps without manual log spelunking.
- Why now: Current logs are too thin to explain real usage, which blocks product iteration on the loop UX.
- Packaging rationale: Telemetry enrichment and deeper log analysis must land together so the kit both records and explains user behavior.
- Launch story: This release upgrades the kit from counting events to understanding operator behavior across a real release session.
