## Current State Analysis
- The committee-driven loop had working research, review capture, and release gates, but its config still treated all committee behavior as three flat roles with no explicit council, secretariat, or evaluator layer.
- That made it hard to strengthen the architecture later without either bloating prompts or overloading the existing roles with too many responsibilities at once.

## Research
- The current config only models three flat committee roles, so it cannot distinguish council membership from secretariat or evaluator responsibilities.
- The next architecture step should add a lightweight V2 roster in config first, while keeping the existing roles array so current rendering and validation do not break.
- A persona-id based roster is enough for this round; detailed field contracts and richer rendering can wait for the next iteration.
