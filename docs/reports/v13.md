## Current State Analysis
- The loop could persist committee review conclusions, but the collected project snapshot and report generator still treated that data as manual re-entry instead of a reusable artifact.
- Without goal-aware reuse, the next report risked either missing the committee rationale or accidentally pulling the wrong iteration's review notes.

## Research
- The project-data snapshot currently omits review_state, so later tools cannot inspect the latest committee-backed rationale from the collected data artifact.
- write-report.py only renders research and committee sections from CLI flags today, which forces manual duplication after capture-review.py has already persisted the same conclusions.
- The loop also needs goal-aware reuse so a previous iteration's review_state cannot leak into a new iteration's report when goals differ.
