## Current State Analysis
- The committee V2 state now had structured research and council lanes, but the actual scope decision for an iteration still had no dedicated durable writer path.

## Research
- The stronger review_state schema now has a research_gate block, but capture-review.py still cannot write structured research summaries, evidence refs, quality scores, or open gaps into it.
- The next step should extend the existing script instead of adding a second capture entrypoint, because operators already know capture-review.py as the durable review writer.
- The compatibility rule is simple: structured research-gate inputs should populate the nested block while the old research bullets continue to work for lightweight flows and reports.
- The new schema can hold council-specific summaries, but capture-review.py still only writes one flat committee_feedback list, so product, architecture, and operator voices remain blended together.
- The next step should add optional council-specific flags without removing the generic committee-feedback flags, because lightweight flows still benefit from the simple path.
- Dissent should be modeled as short structured bullets per council, not as full debate transcripts.
- The council slots now hold differentiated advice, but the actual scope decision for an iteration still has no dedicated writer path, so selected goal and boundaries remain trapped in generic decision bullets or reports.
- The next useful step is to add compact scope-decision flags to capture-review.py instead of inventing a separate scope tool, because the same command already owns durable review writing.
- The state should capture only executable boundaries here: selected goal, why selected, scope in/out, assumptions, risks, required validation, stop conditions, dissent, and next action.
