## Current State Analysis
- The upgraded roster and renderer created a clearer committee architecture, but durable state still had only flat review lists, which made later scope, evaluator, and escalation work hard to model cleanly.
- Without a stronger schema, the next committee upgrades would either overload generic bullet lists or start scattering decision facts across multiple files.

## Research
- The current review_state can store flat findings and decisions, but it has nowhere clean to place council summaries, scope boundaries, evaluator results, or escalation metadata.
- The safest next step is to expand the normalized schema in common.py while keeping the current flat fields alive so report generation and gates keep working during migration.
- A compact nested structure is enough here; the goal is machine-readable decision lanes, not transcript storage.
