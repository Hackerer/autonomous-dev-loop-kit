## Current State Analysis
- The committee-driven kit could be installed into user CLI skill folders, but adoption in a target repository still required manual copying of loop files and references.
- That manual setup path was especially risky around .agent-loop/config.json because target repositories usually need project-specific validation commands that should not be overwritten casually.

## Research
- The repo already has install-to-clis.sh for user-level Codex and Claude skill folders, but it does not provide a repo-local installer for target projects.
- A target-project installer needs to copy committee-driven scripts, references, templates, and skill folders in one step while preserving target-specific loop state by default.
- The highest-risk overwrite is .agent-loop/config.json because target repositories usually need their own validation commands, so config preservation must be the default behavior.
